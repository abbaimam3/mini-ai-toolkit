Explain the primary differences between TensorFlow and PyTorch. When would you choose one over the other?**| Aspect | **TensorFlow** | **PyTorch** ||--------|----------------|-------------|| **Computation Graph** | Uses a static computation graph (TensorFlow 1.x) and later introduced eager execution for flexibility. | Uses a dynamic computation graph that builds at runtime, making debugging intuitive. || **Ease of Use** | Requires more boilerplate but offers powerful production support (TF-Serving, TFX). | Easier for experimentation and research; Pythonic and more flexible for custom models. || **Deployment** | Strong ecosystem for deployment â€” TensorFlow Lite, TensorFlow.js, and TensorFlow Serving. | Deployment tools improving (TorchServe, TorchScript), but less mature than TensorFlowâ€™s. || **Visualization** | TensorBoard offers advanced visualization tools for metrics and model graphs. | Offers TensorBoard integration but less native visualization. || **Preferred Scenario** | When scalability and deployment matter (e.g., large-scale production). | When prototyping and research flexibility matter (e.g., fast experimentation). |**Conclusion:**  Choose **TensorFlow** for production and mobile deployment; choose **PyTorch** for quick research, experimentation, and academic use.---## **Q2. Describe two use cases for Jupyter Notebooks in AI development.**1. **Exploratory Data Analysis (EDA):**     Jupyter allows interactive visualization and code execution, making it ideal for data cleaning, feature engineering, and plotting results step-by-step.2. **Model Prototyping & Documentation:**     Researchers use Jupyter to build, test, and share models interactively with inline visualizations and markdown explanations for reproducibility.---## **Q3.
